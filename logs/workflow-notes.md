### Влияние на результат:

| Изменение               | Наблюдаемый эффект                                            | Вывод                                                                |
| ----------------------- | ------------------------------------------------------------- | -------------------------------------------------------------------- |
| **Короткий t5xxl**      | Изображение общее, мало деталей, архитектура "размытая"       | Flux требует **естественного языка** и полных предложений            |
| **Развёрнутый t5xxl**   | Появилась читаемая перспектива, свет, атмосфера               | Детальное описание = контроль композиции                             |
| **Добавление clip_l**   | Усилились ключевые объекты (кристаллы, свет), меньше "шума"   | `clip_l` работает как "якорь" для важных элементов                   |
| **Guidance 4.5**        | Картинка стала "жёстче", меньше креатива, но точнее к промпту | Для художественных задач лучше 3.5, для точных — 4.5+                |
| **Разрешение 1344×768** | Лучше подходит для вертикальных кадров персонажей             | Использовать под задачу: квадрат — для локаций, 4:3 — для персонажей |

## Лучший вариант и почему

**Выбран:** Вариант 2

**Причины:**
1. Баланс между контролем и креативностью: модель следует промпту, но добавляет интересные детали
2. Читаемая архитектура + атмосфера, соответствующая проекту (минимализм, приглушённые тона)
3. `clip_l` с ключевыми тегами предотвращает "утечку" стиля в посторонние жанры
4. Разрешение 1344×768 даёт кинематографичный кадр

## Шаг 1. Формулировка промптов под архитектуру Flux Dev

**Контекст:**
- Хотел получить стабильный, воспроизводимый визуальный язык для локации «Архив Памяти».
- Для определения структуры запросов, которая будет использоваться во всех дальнейших ассетах проекта.

**Что пробовал:**
- Вариант 1: Теговый промпт (`archive, floating crystals, minimalist, cinematic, 8k`). Модель выдавала абстрактные формы, композиция «плыла», стиль дрейфовал в стандартный sci-fi.
- Вариант 2: Перегруженный поэтичный текст с метафорами. Flux терял архитектурную логику, добавлял нежелательные фантазийные детали, свет становился хаотичным.
- Вариант 3: Структурированный естественный язык. `t5xxl` - полное описание сцены, геометрии, освещения и настроения. `clip_l` - только якорные теги (`brutalist architecture, volumetric lighting, memory shards`).

**Вывод:**
- Принимал Вариант 3 как стандарт для проекта.
- Flux Dev не работает с тегами через запятую: ему нужны связные предложения в `t5xxl`. `clip_l` выполняет роль стилистического ограничителя, а не основного промпта.
- Далее: применить эту схему к генерации персонажей и пропсов, протестировать `guidance=4.0` для усиления контроля над композицией без потери атмосферы.

## Шаг 2. Курация и отбор финальных референсов

**Контекст:**
- Хотели отобрать 4 ключевых изображения, закрывающих визуальные задачи: персонаж, локация, пропс, настроение (распад).
- Для формирования `outputs/selected/` и фиксации базового стиля в GDD.

**Что пробовали:**
- Сгенерировали ~20 вариантов с фиксированным `seed`, меняя `guidance` (2.5 / 3.5 / 4.5) и соотношение сторон.
- Отбросили кадры с «клинической» стерильностью (слишком чисто для геймплея), пережжённым контрастом, нарушением перспективы или уходом в аниме/фэнтези эстетику.
- Оставили изображения, где читаемость пространства/объекта сохраняется даже при добавлении эффектов визуального распада.

**Вывод:**
- Базовым для следующей итерации считаем `02-w04-location-archive-hub.png`. Он задаёт правильные пропорции, световую схему и масштаб, которые легко масштабировать на другие сцены.
- Критерий отбора сместился с «красивой картинки» на «игровую читаемость»: силуэт, ключевые объекты и зоны взаимодействия должны считываться за 1–2 секунды.
- Далее: на основе отобранных 4 кадров провести тесты с ControlNet (Depth) для жёсткой фиксации композиции и подготовить бриф для 3D-белбокса уровня.
