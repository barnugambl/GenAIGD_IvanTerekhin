
## Запись #1 — Название и One-liner
  
**Дата:** 2025-03-18

### Подготовка

**Цель:** Придумать название и elevator pitch для игры.

**Критерии:**

- Название должно отражать тему (смерть, память, воспоминания)

- One-liner должен уместиться в 2-3 предложения

- Нужно 3 варианта на выбор
### Промпт

Придумай 3 варианта названия и one-liner для игры где игрок вселяется в умерших через их воспоминания. Смерть = прогресс. Жанр: Narrative Puzzle-Adventure.

### Результат
| Вариант | Название | One-liner |
|---------|----------|-----------|
| 1 | **Echoes of the Departed** | Игрок проживает смерти разных персонажей, чтобы собрать историю по фрагментам |
| 2 | **Memory of the Dead** | Вселяйся в умерших, узнавай их тайны, разгадывай апокалипсис |
| 3 | **Last Breath Archive** | Архивариус воспоминаний спасает мир через чужие смерти |

### Вывод
**Принято:** Вариант 1 — **Echoes of the Departed**

**Почему:**
-  Звучит поэтично и загадочно
- "Echoes" передаёт идею фрагментарности памяти
- "Departed" мягче чем "Dead", лучше передаёт атмосферу
- Вариант 2 — слишком буквально
- Вариант 3 — слишком длинно

## Запись #2 — Core Loop

**Дата:** 2025-03-18

### Подготовка
**Цель:** Определить основной игровой цикл.
**Критерии:**
- Цикл должен быть понятным за 30 секунд
- Включать смерть как механику
- 4-6 шагов максимум

### Промпт
Создай 2 версии core loop для игры где смерть = прогресс. Игрок вселяется в умерших и проживает их последние моменты.

### Результат
**Версия A (классическая):**
1. Вселение → 2. Исследование → 3. Смерть → 4. Анализ → 5. Прогресс → 6. Новое вселение

**Версия B (с выбором):**
1. Выбор фрагмента → 2. Проживание → 3. Смерть/Финал → 4. Синтез подсказок → 5. Открытие новых путей

### Вывод
**Принято:** Синтез обеих версий

**Финальный core loop:**
1. **Вселение** — выбор фрагмента памяти из архива
2. **Исследование** — проживание последних минут жизни
3. **Смерть** — неизбежный финал (скрипт или ускорение)
4. **Анализ** — открытие "дерева подсказок"
5. **Синтез** — сбор фрагментов в единую картину
6. **Прогресс** — открытие новых воспоминаний

**Почему синтез:**
- Версия A — чёткая структура
- Версия B — акцент на выборе и синтезе
- Чистая Версия A — слишком линейно
- Чистая Версия B — менее понятна для новичков

## Запись #3 — 3 Pillars

**Дата:** 2025-03-18

### Подготовка
**Цель:** Сформулировать 3 опоры проекта.
**Критерии:**
- Каждая опора — 1-2 предложения
- Должны быть уникальными для этого проекта
- Не общие фразы типа "fun gameplay"

### Промпт
Создай 3 pillars для игры про воспоминания умерших. Каждая опора должна отражать уникальную механику или философию. Избегай клише вроде "интересный геймплей".

### Результат
**Варианты от ИИ:**
1. Смерть как открытие / Повествование через фрагменты / Эмоциональный вес
2. Инструментальная смерть / Нелинейная история / Атмосфера утраты
3. Умри чтобы узнать / Собери пазл / Почувствуй трагедию

### Вывод
**Принято (финальные формулировки):**

| # | Опора | Формулировка |
|---|-------|--------------|
| 1 | **Смерть как инструмент** | Смерть не наказывает, а награждает информацией. Игрок хочет умирать, чтобы узнать больше. |
| 2 | **Нелинейное повествование** | История собирается как пазл из разных перспектив. Игрок сам строит гипотезы. |
| 3 | **Эмоциональный вес** | Каждая смерть — чья-то трагическая история. Игрок чувствует ответственность. |

**Почему:**
- Коротко и понятно
- Каждая опора уникальна для этого проекта
- Отражают геймплей + нарратив + эмоции
- Отброшены варианты с клише ("атмосфера утраты" — слишком размыто)
